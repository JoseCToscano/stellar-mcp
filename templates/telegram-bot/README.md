# Stellar MCP — Telegram Bot Template

A Telegram bot that connects to any **Stellar MCP server** and lets users interact with Soroban smart contract functions — directly in Telegram.

Built with [grammy](https://grammy.dev) + [`@stellar-mcp/client`](https://github.com/JoseCToscano/stellar-mcp).

---

## What it does

- `/tools` — dynamically lists all functions from your contract
- `/call` — browse tools via inline keyboard, see argument previews, call tools
- `/call <tool> <json-args>` — execute any contract function directly
- **AI chat mode** (optional) — type naturally; the LLM calls contract tools on your behalf

The bot works with **any** MCP server generated by `stellar mcp generate`. You only change the `MCP_SERVER_URL` environment variable.

## Quick start (local, ~5 minutes)

### 1. Clone and install

```bash
cp -r templates/telegram-bot my-stellar-bot
cd my-stellar-bot
pnpm install
```

### 2. Create a Telegram bot

1. Open Telegram, message [@BotFather](https://t.me/BotFather)
2. Send `/newbot` and follow the prompts
3. Copy the bot token

### 3. Configure environment

```bash
cp .env.example .env
```

Edit `.env`:

```bash
TELEGRAM_BOT_TOKEN=<your-bot-token>
MCP_SERVER_URL=<your-mcp-server-url>
SIGNER_SECRET=<stellar-secret-key>        # optional — omit for read-only
NETWORK_PASSPHRASE=Test SDF Network ; September 2015
RPC_URL=https://soroban-testnet.stellar.org
```

**Test with the Token Factory contract:**
```bash
MCP_SERVER_URL=https://stellar-mcp-token-factory.vercel.app/mcp
```

### 4. Run locally

```bash
pnpm dev
```

Open Telegram and message your bot:
- `/start` — welcome message
- `/tools` — list contract tools
- `/call` — interactive tool browser
- `/call get-admin` — call a read-only tool directly

## AI chat mode

Set these in `.env` to enable natural language interaction:

```bash
AI_ENABLED=true
ANTHROPIC_API_KEY=<your-key>    # recommended
# or
OPENAI_API_KEY=<your-key>
```

Then just chat:
```
You: what's the admin of this contract?
Bot: The admin address is GABC...XYZ

You: how many tokens have been deployed?
Bot: 42 tokens have been deployed so far.

You: deploy a token called "Solar" with symbol "SOL" and 7 decimals
Bot: ✅ Transaction submitted!
     Hash: abc123...
     Status: SUCCESS
```

The LLM automatically selects and calls the right contract tools based on your message.

> **Note:** Conversation history is stored in memory and resets when the server restarts. For persistent history, see the Vercel KV section below.

## Deploy to Vercel

### 1. Install Vercel CLI

```bash
npm i -g vercel
```

### 2. Deploy

```bash
vercel deploy
```

### 3. Set environment variables

In the Vercel dashboard → Project → Settings → Environment Variables, add all variables from your `.env`.

Or via CLI:

```bash
vercel env add TELEGRAM_BOT_TOKEN
vercel env add MCP_SERVER_URL
vercel env add SIGNER_SECRET
vercel env add NETWORK_PASSPHRASE
vercel env add RPC_URL
```

### 4. Register the webhook

After deployment, register your Vercel URL as the Telegram webhook:

```bash
VERCEL_URL=https://your-app.vercel.app
BOT_TOKEN=<your-bot-token>

curl "https://api.telegram.org/bot${BOT_TOKEN}/setWebhook?url=${VERCEL_URL}/api/webhook"
```

You should see: `{"ok":true,"result":true}`

### Verify it's working

```bash
curl "https://api.telegram.org/bot${BOT_TOKEN}/getWebhookInfo"
```

## Customization

### Use a different contract

Change `MCP_SERVER_URL` to point to any running MCP server. The bot discovers tools dynamically — no code changes needed.

### Use a different AI model

Edit `src/ai.ts` — the `getModel()` function:
```typescript
// Use Claude Sonnet instead of Haiku
return anthropic('claude-sonnet-4-6');

// Use GPT-4o mini
return openai('gpt-4o-mini');
```

### Persistent conversation history (production)

Install Vercel KV from the Vercel marketplace, then replace `src/history.ts` with:

```typescript
import { kv } from '@vercel/kv';
import type { CoreMessage } from 'ai';

const TTL = 60 * 60 * 24; // 24 hours

export async function getHistory(chatId: number): Promise<CoreMessage[]> {
  return (await kv.get<CoreMessage[]>(`chat:${chatId}`)) ?? [];
}

export async function appendHistory(chatId: number, role: 'user' | 'assistant', content: string): Promise<void> {
  const history = await getHistory(chatId);
  history.push({ role, content });
  if (history.length > 20) history.splice(0, history.length - 20);
  await kv.setex(`chat:${chatId}`, TTL, history);
}

export async function clearHistory(chatId: number): Promise<void> {
  await kv.del(`chat:${chatId}`);
}
```

## Environment variables reference

| Variable | Required | Description |
|----------|----------|-------------|
| `TELEGRAM_BOT_TOKEN` | Yes | From @BotFather |
| `MCP_SERVER_URL` | Yes | Your MCP server URL |
| `SIGNER_SECRET` | No | Stellar secret key — enables write operations |
| `NETWORK_PASSPHRASE` | Yes | Stellar network passphrase |
| `RPC_URL` | Yes | Soroban RPC endpoint |
| `AI_ENABLED` | No | `true` to enable AI chat mode |
| `ANTHROPIC_API_KEY` | No | Enables Claude (recommended for AI mode) |
| `OPENAI_API_KEY` | No | Enables GPT-4o (alternative for AI mode) |

## Troubleshooting

**Bot doesn't respond**
- Check `pnpm dev` output for errors
- Verify `TELEGRAM_BOT_TOKEN` is correct
- Ensure `MCP_SERVER_URL` is reachable

**"Could not reach MCP server"**
- Make sure your MCP server is running
- Check that `MCP_SERVER_URL` includes the full path (e.g. `.../mcp`)

**Webhook not receiving updates (Vercel)**
- Confirm webhook is set: `getWebhookInfo` should show your URL
- Check Vercel function logs in the dashboard
- Vercel function URL must be HTTPS

**AI mode not working**
- Ensure `AI_ENABLED=true` in your environment
- Verify your API key is valid
- Check Vercel function logs for rate limit or quota errors

**Write operations not submitting**
- Ensure `SIGNER_SECRET` is a valid Stellar secret key (starts with `S`)
- The account must be funded on the correct network
- Check that `NETWORK_PASSPHRASE` matches your `RPC_URL`
